Set up the Agent Hub Dashboard integration for this OpenClaw workspace. This connects your agents to a live web dashboard that displays their status, activity, cron health, todos, and blockers in real time.

Before starting, explore this workspace — look at the agents/ directory structure, read any SOUL.md files you find, and check ~/.openclaw/cron/jobs.json to understand what cron jobs are active.

---

YOUR TASKS:

1. CREATE scripts/dashboard-heartbeat.py

Write a Python 3 script (stdlib only — no pip installs, use urllib not requests) that does the following every time it runs:

- Loads Supabase credentials from .env.dashboard in the workspace root
- Auto-discovers all agents by scanning agents/*/SOUL.md — parse each agent's name (first # heading), emoji (first emoji character), purpose (## Purpose or ## Identity section), and model field
- Reads cron job status from ~/.openclaw/cron/jobs.json
- Reads manual overrides from agents/known-issues.json if it exists
- Pushes everything to Supabase using the REST API (service role key)

Supabase REST pattern: POST {SUPABASE_URL}/rest/v1/{table} with headers:
  apikey: {SUPABASE_SERVICE_KEY}
  Authorization: Bearer {SUPABASE_SERVICE_KEY}
  Content-Type: application/json
  Prefer: resolution=merge-duplicates,return=representation  (for upserts that need IDs back)

Tables to write to:

agents — upsert by name:
  name, emoji, purpose, model, is_active=true, last_heartbeat_at (ISO timestamp), heartbeat_payload (jsonb)
  heartbeat_payload should contain: status (active/idle/degraded/blocked/errored), heartbeat_source="openclaw-cron",
  last_output_hours_ago (check recency of files in agents/{dir}/output/),
  degraded_reasons (list of erroring cron names if any), healthy_crons (list of healthy cron names),
  and any fields from known-issues.json overrides (tools, output_type, revenue_model, blockers, etc.)
  Status logic: start from cron health (erroring = degraded, ok = active, no crons = idle),
  then overlay known-issues.json if the override is more severe.

cron_jobs — upsert by name:
  name, agent_id (FK — look up from agents upsert result), schedule, expected_cadence_minutes, last_run_at, next_expected_at, enabled=true

agent_activity — insert only (append log, do not upsert):
  agent_id (FK), event_type="log", payload={"message": "Heartbeat — status: {status}"}

After upserting agents use return=representation to get back the UUIDs — you need them for agent_id FK fields in other tables.

2. CREATE .env.dashboard in the workspace root:

SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co
SUPABASE_SERVICE_KEY=YOUR_SERVICE_ROLE_KEY_HERE

Tell the user: get these from Supabase → Settings → API. SUPABASE_SERVICE_KEY is the service_role key (not the anon key).

3. CREATE agents/known-issues.json if it does not already exist:

{"_doc": "Manual agent status overrides the heartbeat cannot auto-detect.", "agent_issues": {}}

4. REGISTER the cron job using the actual absolute path to this workspace:

openclaw cron add --name "📊 Dashboard Heartbeat" --cron "*/15 * * * *" --target isolated --model haiku --message "Run this command: python3 {ABSOLUTE_WORKSPACE_PATH}/scripts/dashboard-heartbeat.py"

5. RUN the script once manually to verify it connects to Supabase and writes data. Show the output. If credentials are missing, skip and remind the user to fill in .env.dashboard first.
